# MyGradeSystem - 成绩查询与管理系统

## 项目简介

MyGradeSystem 是一款基于 **Java Web** 技术栈开发，并部署在 **AWS云平台** 上的B/S架构成绩管理系统。它旨在解决传统成绩管理中效率低下、易出错和权限模糊等问题，通过清晰的**学生、教师、管理员**三级角色划分，实现了成绩从录入、查询到管理的全流程数字化。

## 系统功能概览

### 👨‍🎓 学生角色
- **个人信息查看**：查看学号、姓名、班级等基本信息。
- **成绩查询与统计**：查询个人成绩，查看已修课程、总学分、平均绩点等统计信息。
- **查询时间控制**：仅在教师设定的时间范围内方可查询成绩。
- **密码修改**：自助修改登录密码。

### 👨‍🏫 教师角色
- **成绩管理**：对任教课程的学生成绩进行增、删、改、查。
- **批量导入**：通过CSV/Excel模板批量导入成绩，系统提供详细的数据校验与错误反馈。
- **系统配置**：设定学生可查询成绩的时间范围。
- **密码修改**：自助修改登录密码。

### 👨‍💼 管理员角色
- **用户管理**：管理学生和教师的基础信息（增、删、改、查）。
- **密码重置**：为用户重置遗忘的密码至初始状态。

## 技术架构

### 后端技术栈
- **Java**：核心编程语言。
- **Servlet & JSP**：处理Web请求和动态页面渲染。
- **JDBC**：用于应用程序与数据库的连接和交互。
- **Maven**：项目构建与依赖管理。

### 前端技术栈
- **HTML, CSS, JavaScript**：构建用户界面和实现交互逻辑。

### 数据库
- **MySQL**：关系型数据库，用于存储所有业务数据。

### 部署环境 (AWS)
- **EC2**：云服务器，用于部署Tomcat和应用。
- **RDS**：托管式MySQL数据库服务，实现高可用。
- **VPC**：虚拟私有云，通过公有/私有子网隔离，保障网络安全。
- **安全组**：充当虚拟防火墙，严格控制访问权限。

## 部署指南

### 前提条件
1. 一个AWS账户及相应的资源权限。
2. 本地开发环境：JDK 17+、Maven、IDEA。
3. 远程连接工具：Xshell（或其它SSH客户端）、Xftp（或其它FTP/SFTP客户端）。

### AWS环境搭建
1. **创建VPC网络**
   - 创建VPC，CIDR设置为 `10.0.0.0/16`。
   - 在两个可用区创建公有子网和私有子网。
   - 配置互联网网关与NAT网关，确保网络连通性与安全性。

2. **创建RDS数据库**
   - 在私有子网中创建MySQL实例（建议使用多可用区部署以保障高可用）。
   - 创建对应的数据库安全组，仅允许来自EC2安全组的3306端口访问。
   - 记录数据库的终端节点（Endpoint）、端口、用户名和密码。

3. **创建EC2实例**
   - 选择Amazon Linux 2023 AMI及合适的实例类型（如t3.small）。
   - 将其部署在公有子网中，并自动分配公网IP。
   - 绑定密钥对（.pem文件）以便SSH远程连接。
   - 关联Web安全组，开放22（SSH）和8080（Tomcat）端口。

### 应用部署步骤
1. **配置EC2环境**
   - 通过SSH连接到EC2实例。
   - 安装Java 17环境：`sudo yum install -y java-17-amazon-corretto-devel`
   - 使用Xftp将Tomcat压缩包上传至EC2，并解压到 `/usr/local/tomcat` 目录。
   - 配置Tomcat用户、权限及systemd服务（详细命令见项目报告）。

2. **构建与上传应用**
   - 在本地，使用Maven将项目打包成WAR文件：`mvn clean package`。
   - 使用SCP命令或Xftp将WAR文件上传到EC2实例。
   - 将WAR文件移动到Tomcat的 `webapps` 目录下。

3. **启动服务**
   - 启动Tomcat服务：`sudo systemctl start tomcat`。
   - 在浏览器中访问 `http://<您的EC2公网IP>:8080/MyGradeSystem/` 来验证应用是否正常运行。

## 使用说明
1. 使用不同角色的账号登录系统。
2. 学生和教师首次登录后请及时修改默认密码。
3. 教师在上传成绩前，请务必下载并使用系统提供的标准模板。
4. 管理员负责维护用户基础信息和处理密码重置请求。

## 注意事项
- 部署时，请务必检查AWS安全组规则，确保端口的正确开放与源IP的限制。
- RDS数据库的终端节点信息需正确配置在项目的数据库连接文件中。
- 本系统为课程项目，在生产环境使用时建议进一步加强安全性配置，如使用HTTPS、更严格的数据库访问策略等。
